Why I built this ↓

Why I built this

It is March 2026. No one should be spending their time hand-making slide decks anymore. Coming up with what you want to say takes 5% of the time. The other 95% is dragging elements around.

The alternatives don't work

The AI slide tools that exist are insufficient. Google Slides' built-in AI is bad. Gamma and platforms like it are polished — but they pull you into yet another system, default to cliché AI-generated imagery with no way to tune it out, and add another subscription to the pile. I tried them. I didn't want to migrate my team or re-explain my product and brand from scratch.

I also tried having Claude Code output PowerPoint files directly. Those don't come out well either.

Why PowerPoint fails AI

PowerPoint was built for humans who can't write code. It is a GUI — a control panel of drag handles and buttons so that we can manipulate layout without touching the underlying logic. When you ask Claude to work in PowerPoint, you are asking it to operate through a human-facing abstraction layer.

AI doesn't need the GUI. It is the coder.

It has to translate: intent → GUI operation → output. Every step in that chain loses fidelity. HTML and CSS skip the middleman entirely. The code is the output. Claude already knows my product, my design system, my value prop. I go from brainstorming to a finished deck in minutes — without switching tools or re-explaining anything.

This is probably a broader pattern. Tools built to simplify things for humans make things harder for AI. The abstraction that helps us becomes friction for them.

Two problems remained

HTML slides don't go fullscreen by default. And on a bigger projector, the aspect ratio gets off and you're fumbling with browser zoom mid-presentation.

So I built this. It took a few minutes using Claude Code. Drop in any HTML file, pick your ratio, go fullscreen. That's it.
