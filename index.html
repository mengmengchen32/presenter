Manifesto

It is March 2026. No one should be hand-making slide decks anymore. Coming up with what you want to say takes 5% of the time. The other 95% is dragging elements around.

The alternatives don't work

The AI slide tools that exist are insufficient. Google Slides' built-in AI is bad. Gamma and platforms like it are polished — but they pull you into a new system, force you to re-explain your product and brand from scratch, default to cliché AI-generated imagery with no way to turn it off, and add another subscription to the pile.

I tried having Claude Code output PowerPoint files directly. Those don't work either.

Why PowerPoint fails AI

PowerPoint was built for humans who can't write code. It is a GUI — a control panel of drag handles and buttons so that we can manipulate layout without touching the underlying logic.

When you ask Claude to work in PowerPoint, you are asking it to operate through a human-facing abstraction layer. It has to translate: intent → GUI operation → output. Every step in that chain loses fidelity.

AI doesn't need the GUI. It is the coder.

HTML and CSS skip the middleman entirely. The code is the output. Claude already knows my product, my design system, my value prop. I go from brainstorming to a finished, beautiful deck in minutes — without switching tools or re-explaining anything.

This is probably a broader pattern. Tools built to simplify things for humans make things harder for AI. The abstraction that helps us becomes friction for them.

Two small problems remained

HTML slides don't go fullscreen by default. And on a bigger projector, the aspect ratio goes off and you're fumbling with browser zoom mid-presentation.

So I built Presenter. It took a few minutes using Claude Code. Drop in any HTML file, pick your aspect ratio, go fullscreen. That's it.

Now we can all start living in a world where we don't have to make slides by hand anymore.
